# 固件管理应用设计方案

## 一、项目概述

### 1.1 项目目标
开发一款用于Rockchip Android 14设备的系统级固件升级应用，支持以下功能：
- 服务器地址配置（扫码/手动输入）
- 固件版本检查和更新
- 断点续传下载
- 免解压升级
- 系统更新应用
- 重启管理

### 1.2 技术栈
- 语言：Kotlin
- UI框架：Jetpack Compose
- 网络库：OkHttp
- 扫码库：zxing-android-embedded
- 最低SDK：API 30 (Android 11)
- 目标SDK：API 36 (Android 14+)

## 三、设计模式

### 3.1 Singleton Pattern (单例模式)
**应用位置**: PrefsManager, TaskStateManager, UpdateEngineWrapper

```kotlin
// 例如 PrefsManager
object PrefsManager {
    private lateinit var prefs: SharedPreferences
    
    fun init(context: Context) {
        prefs = context.getSharedPreferences("ota_prefs", MODE_PRIVATE)
    }
    
    fun getServerBaseUrl(): String? = prefs.getString("server_url", null)
    fun setServerBaseUrl(url: String) {
        prefs.edit().putString("server_url", url).apply()
    }
}
```

**优点**:
- 全局唯一访问点
- 延迟初始化
- 资源共享

---

### 3.2 State Pattern (状态模式)
**应用位置**: MainActivity中使用Stage状态管理

```kotlin
enum class Stage {
    IDLE,                // 空闲状态
    CHECK_PREPARE,       // 检查更新
    DOWNLOADING,         // 下载中
    DOWNLOADED,          // 下载完成
    PREPARING,           // 准备升级
    APPLYING,            // 应用升级
    REBOOT_PENDING,      // 等待重启
    ERROR                // 错误状态
}
```

**优点**:
- 状态转换清晰
- 易于添加新状态
- 避免大量if-else

### 3.5 Observer Pattern (观察者模式)
**应用位置**: UpdateEngineCallbackAdapter

```kotlin
interface UpdateEngineListener {
    fun onStatusUpdate(status: Int, percent: Float)
    fun onPayloadApplicationComplete(errorCode: Int)
    fun onPayloadApplicationSuccess()
    fun onPayloadApplicationError(errorCode: Int, message: String)
}

abstract class UpdateEngineCallbackAdapter {
    private var listener: UpdateEngineListener? = null
    
    fun setListener(listener: UpdateEngineListener?) {
        this.listener = listener
    }
    
    open fun onStatusUpdate(status: Int, percent: Float) {
        listener?.onStatusUpdate(status, percent)
    }
}
```

## 四、核心功能实现

### 4.1 五阶段状态机流程

```
IDLE -> CHECK_PREPARE -> DOWNLOADING -> PREPARING -> APPLYING -> REBOOT_PENDING
  ↓         ↓             ↓            ↓           ↓            ↓
  └─────────┴─────────────┴────────────┴───────────┴────────────┴──→ ERROR
```

**状态说明**:
1. **IDLE**: 初始状态，等待用户操作
2. **CHECK_PREPARE**: 请求服务器检查更新
3. **DOWNLOADING**: 下载固件（支持断点续传）
4. **PREPARING**: 免解压解析payload信息
5. **APPLYING**: 应用升级到系统
6. **REBOOT_PENDING**: 升级完成，等待重启

## 五、设计原则遵循

### 5.1 SOLID原则

| 原则 | 说明 | 应用位置 |
|------|------|----------|
| **S** - Single Responsibility | 每个类只有一个职责 | 所有工具类 |
| **O** - Open/Closed | 对扩展开放，对修改关闭 | Stage枚举、UpdateEngineListener接口 |
| **L** - Liskov Substitution | 子类可以替换基类 | UpdateEngineCallbackAdapter |
| **I** - Interface Segregation | 接口最小化 | UpdateEngineListener |
| **D** - Dependency Inversion | 依赖抽象而非具体 | 依赖注入 |

### 5.2 其他最佳实践

- **KISS原则**: Keep It Simple, Stupid - 避免过度设计
- **DRY原则**: Don't Repeat Yourself - 代码复用
- **Separation of Concerns**: 视图、逻辑、数据分离
- **Fail Fast**: 尽早检测和报告错误
